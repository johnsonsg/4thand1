# PR Process

## Branching

- Create a feature branch off `main`.
- Keep branches focused and small.

## Development

- Run `npm run dev` from repo root.
- Add/adjust tests if applicable.
- Update docs when behavior changes.

## Before Opening a PR

- `npm run lint`
- `npm run build`
- Verify key flows locally (public site + admin if touched)

## PR Checklist

- Clear title and summary
- Screenshots for UI changes
- Notes on configs or env vars
- Link to related issue or ticket (if any)
